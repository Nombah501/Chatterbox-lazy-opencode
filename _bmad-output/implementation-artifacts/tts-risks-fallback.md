# Риски TTS и план fallback

## Основные риски
- **Тяжелые зависимости**: torch/torchaudio могут усложнить установку.
- **GPU недоступен**: пользователь ожидает GPU, но доступен только CPU.
- **Большая задержка**: первая загрузка модели и генерация аудио.
- **Ошибки голос-клона**: некорректный `voice_prompt_path` или неподдерживаемый файл.
- **Ресурсные ограничения**: нехватка VRAM/CPU, падения в рантайме.

## Стратегия fallback
- **Device auto**: использовать GPU при наличии, иначе CPU с предупреждением.
- **Graceful degrade**: при ошибке TTS — текстовый вывод остается полным.
- **Валидация параметров**: привести к дефолтам, не падать.
- **Таймаут/ограничение**: опционально ограничить время озвучки (на будущее).

## Диагностика
- Печатать краткое предупреждение при переходе на CPU.
- Логировать ошибку TTS в диагностический вывод.
- Не логировать секреты или пути, если это нежелательно.

## Рекомендации по запуску
- Для GPU: указать `--tts-device cuda`.
- Для CPU: `--tts-device cpu` и ожидать более долгую генерацию.
- Для voice prompt: использовать WAV/короткий референс.
