# Product Brief: Chatterbox lazy opencode

<!-- Content will be appended sequentially through collaborative workflow steps -->

## Executive Summary

Chatterbox lazy opencode — это open-source инструмент для OpenCode CLI, который синтезирует ответы нескольких ИИ‑агентов в структурированный результат (краткий итог + расхождения + цитаты/источники) и при необходимости озвучивает итог через TTS. Продукт вдохновлен AgentVibes для Claude Code, но фокусируется на OpenCode‑экосистеме и на управляемом, минимальном ядре. Основная аудитория — соло‑разработчики, проводящие много часов в OpenCode и готовые улучшать опыт работы за счет аудио‑слоя и более прозрачной структуры вывода.

---

## Core Vision

### Problem Statement

Пользователи OpenCode получают ответы ИИ‑агентов только в текстовом виде и не имеют нативного, контролируемого аудио‑сопровождения. Это ограничивает способы восприятия результатов и не использует потенциал TTS для вариативности и доступности.

### Problem Impact

Для соло‑разработчиков, которые проводят много часов в OpenCode, отсутствие аудио‑слоя означает более монотонный опыт и невозможность переключать внимание без потери контекста. Это не критическая боль, но ощутимая возможность улучшить UX и доступность.

### Why Existing Solutions Fall Short

- Общие TTS‑инструменты не интегрированы в OpenCode и не учитывают формат вывода ИИ‑агентов.
- AgentVibes ориентирован на Claude Code и не предоставляет нативного пути для OpenCode.
- Без стандарта «итог/расхождения/цитаты» текстовый поток быстро становится шумным и сложным для восприятия.

### Proposed Solution

Тонкий CLI‑core для OpenCode, который:
- агрегирует ответы ИИ‑агентов;
- формирует структурированный результат: краткий итог + расхождения + прозрачные цитаты/источники;
- опционально озвучивает итог через TTS (Chatterbox) по событию;
- сохраняет вывод в файлы (txt/md) для повторного использования.

### Key Differentiators

- OpenCode‑first интеграция через CLI и плагины.
- Формат вывода «итог + расхождения» как стандарт UX.
- Прозрачные источники и цитаты для проверки синтеза.
- Опциональная TTS‑озвучка по событию, без шума в процессе.
- Модульная архитектура: CLI‑ядро + отдельный TTS‑воркер.
